# 场景拆分（Scene Planning）

把脚本切成节奏均匀、信息密度合适的场景序列。

## 时长与帧数换算

Remotion 以"帧"为单位。默认 fps = 30：

```
durationInFrames = 秒数 × fps
```

| 秒数 | 帧数 (fps 30) |
|------|---------------|
| 3s   | 90            |
| 5s   | 150           |
| 8s   | 240           |
| 10s  | 300           |

单场景建议 3–8 秒；**不要超过 10 秒**，避免观看疲劳。

## 拆分方法

1. 通读文案，按"一个信息点 = 一个场景"切分。
2. 标注每场景：类型（标题 / 要点列表 / 图标功能 / 数据卡 / 结语）、核心文字、建议时长。
3. 计算每个场景的 `startFrame`（上一场景结束帧）与 `durationInFrames`。
4. 在 `src/Video.tsx` 用 `<Sequence from={start} durationInFrames={dur}>` 包裹对应场景组件。

## 场景类型模板

| 类型 | 推荐组件 | 内容 |
|------|----------|------|
| 标题/封面 | `TitleCard` | 主标题 + 副标题 |
| 要点列表 | `BulletList` | 3–5 条带图标要点 |
| 功能展示 | `IconRow` | 3 个图标 + 标签 |
| 字幕旁白 | `Caption` | 底部字幕条 |
| 数据卡 | 自定义 | 大数字 + 说明 |
| 结语/CTA | `TitleCard` | 总结 + Logo/关注 |

## 转场设计

- 统一风格：每个场景进入用 `opacity 0→1`（前 8–15 帧）+ `translateY 40→0`（spring）。
- 避免叠化过长；场景间可留 0–6 帧黑场或无缝衔接。
- 示例见 `assets/remotion-template/src/components/ui.tsx` 中各组件的动画写法。

## 节奏自检

- 总时长 = Σ 各场景时长。若超用户预期，压缩最长场景而非均匀砍。
- 信息密度：竖屏一屏不超过 5 行文字，字号够大。
