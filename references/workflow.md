# 工作流详解：从文字到视频

本文件补充 SKILL.md 第 1 步，给出"动手前必须确认"的信息清单与整体节奏。

## 1. 动手前先对齐（必做）

向用户确认以下信息，缺失则主动追问或给出合理默认：

| 项目 | 说明 | 默认 |
|------|------|------|
| 文案/主题 | 用户要表达的文字内容。可让模型基于主题扩写成口播稿 | 无（必须） |
| 视频时长 | 总时长秒数 | 由场景数推导（每场景 3–8s） |
| 投放平台 | 决定尺寸 | 抖音竖屏 |
| 尺寸 | 9:16 / 16:9 / 3:4 / 21:9 | 1080×1920 (9:16) |
| BGM | 是否要背景音乐 | 可选，默认不加或加轻氛围乐 |
| 风格 | 科技暗色 / 简约亮色 / 品牌色 | 科技暗色渐变 |

## 2. 平台与尺寸对照

| 平台 | 比例 | 分辨率 (宽×高) | fps |
|------|------|----------------|-----|
| 抖音 / 视频号 | 9:16 | 1080×1920 | 30 |
| YouTube / B站 | 16:9 | 1920×1080 | 30 |
| 小红书 | 3:4 | 1080×1440 | 30 |
| 宽屏演示 | 21:9 | 2560×1080 | 30 |

在 `src/Root.tsx` 的 `<Composition>` 里设置 `width`/`height`/`fps`；`durationInFrames` = 总秒数 × fps。

## 3. 整体节奏建议

- 开场（Hook）：1 个标题场景，≤ 5s，抓注意力。
- 主体：每点一个场景，3–8s。
- 结尾：总结/Logo/CTA，3–5s。
- 转场统一：opacity + 轻微 translateY（见 `visual-style.md`），不要每个场景用不同花哨转场。

## 4. 输出前的自检清单

- [ ] 文案逐字校对无错别字
- [ ] 每个场景时长与时长总和一致（`durationInFrames`）
- [ ] 字号在小屏可读
- [ ] BGM 已获授权且已放入 `public/`（如启用）
- [ ] 已告知用户预计渲染时长
